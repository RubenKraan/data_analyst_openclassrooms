Projet 5 – DATAImmo : Modélisation d’une base de données immobilière

Durée : 60 heures

Pourquoi ce projet ?

DATAImmo est un projet stratégique de Laplace Immo, un réseau national d’agences immobilières. La CTO, Clara Daucourt, m’a confié la refonte du modèle de données pour intégrer les transactions foncières, les recensements INSEE et le référentiel géographique français — tout en garantissant la conformité RGPD.

En quoi ces compétences sont importantes

Créer, normaliser et maintenir une base relationnelle ; assurer son intégrité ; rédiger des scripts SQL : ces activités sont au cœur du métier de Data Analyst et BI Analyst.

Méthodologie

  1. Dictionnaire de données
    • Template fourni : code, type, longueur, règles RGPD
    • Remplissage pour DVF, INSEE, géographie
  2. Schéma relationnel
    • Adaptation UML en 3NF pour intégrer population et régions
    • Justification des clés primaires/étrangères
  3. Implémentation SGBD
    • Choix de SQLite Studio
    • Script SQL CREATE / chargement CSV
    • Vérification d’intégrité & volume de données
  4. Validation & présentation
    • Document technique complet
    • Workshop de validation avec la CTO

Résultats

  • Dictionnaire RGPD-compliant (Excel)
  • Schéma UML exporté (PNG)
  • Script SQL commenté (create_dataimmo.sql)
  • Présentation de validation (PDF)

Outils & technologies

  • SQLite Studio
  • Excel & Power Query
  • Visual Studio Code

Compétences démontrées

  • Conception et normalisation de bases relationnelles
  • Implémentation et contrôle d’intégrité des données
  • Documentation technique et animation de workshop
